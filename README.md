# product-sales-sql-case-study
This project contains a SQL case study analysis performed on a dataset consisting of three tables: FactTable, ProductTable, and LocationTable. The goal of this project is to practice and demonstrate different SQL concepts used in data analysis.

In this project, multiple SQL queries were written to analyze sales, profit, marketing expenses, product performance, and regional data. The queries involve retrieving, filtering, aggregating, and transforming data to extract meaningful insights.

Various SQL operations were performed such as data retrieval, aggregation, joining tables, ranking, conditional analysis, and data manipulation.


SQL Concepts Used

Aggregate Functions: SUM(), AVG(), MIN(), MAX(), COUNT()

Filtering and Conditions: WHERE, BETWEEN, CASE WHEN

Sorting and Grouping: ORDER BY, GROUP BY, HAVING

Joins: INNER JOIN

Window Functions: DENSE_RANK()

Advanced SQL: ROLLUP, UNION, INTERSECT

Database Objects: STORED PROCEDURE, USER DEFINED FUNCTION

Data Manipulation: UPDATE, DELETE

String Functions: SUBSTRING(), ASCII()

project helps in strengthening SQL skills required for data analysis and working with relational databases.
